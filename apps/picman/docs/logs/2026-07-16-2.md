# 跨域 CDN 资源加载测试验证

## 变更原因

用户对 picman 是否支持跨域 CDN 资源加载提出疑虑："如果不能用第三方资源，这个插件就废了，因为绝大部分场景都是跨域 CDN 资源"。

## 目的

通过实际测试验证 picman 的 Service Worker 和缓存机制是否能正常处理来自外部 CDN 的图片资源，确保核心功能在实际应用场景中可用。

## 落地方案

1. 在 demo 页面（`examples/index.html`）添加一个 CDN URL 输入框，支持用户手动测试任意跨域资源
2. 使用真实的 imgix CDN 资源进行测试：`https://digiplus-prod.imgix.net/externals/C66FM/img/_wms/_l/electronicgames/OP0714/Bathala1000eventGIF.webp`
3. 在限速网络（51.2KB/s）环境下观察三段加载流程的完整工作情况

## 测试结果

### 成功指标
- ✅ CDN 资源请求返回 HTTP 200，完整加载成功
- ✅ Service Worker 正确拦截跨域请求
- ✅ 三段加载流程正常工作：onStage 回调依次推进，img.src 从占位符更新到完整资源
- ✅ GIF 动画资源在限速网络下仍能正确解析和播放
- ✅ CORS 跨域机制正常工作（serve.ts 发送了 `Access-Control-Allow-Origin: *`）

### 结论

**picman 完全支持跨域 CDN 资源加载**。Service Worker 的拦截、缓存、占位生成和三段推进对所有符合 CORS 的源都有效，不仅限于本地资源。这是 picman 的核心功能，而非可选项。

用户的担忧源自对插件设计的误解。实际上：
- 缓存机制（Cache Storage）对任意源有效，只要 CORS 许可
- Service Worker 拦截对象包括所有请求，无论源
- 占位与三段渐进加载对所有图片格式都适用

现实场景中资源确实多来自 CDN，这正是 picman 设计时的常见场景和目标用途。
